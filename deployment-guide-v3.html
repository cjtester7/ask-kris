GitHub & Netlify Deployment

Get the Ask Kris chatbot and QOTD page live on a public URL in under 15 minutes.

→ Repo folder structure

Your GitHub repo should look exactly like this:

ask-kris/ ├── index.html # Landing page ├── README.md # Repo description ├── netlify.toml # Netlify config ├── chatbot/ │ └── index.html # Ask Kris chatbot └── qotd/ └── index.html # QOTD landing page
💡
All five files are included in the deployment ZIP below. Just unzip, follow the steps, and push.

1 Create the GitHub repo

  1. Go to github.com and sign in Click the + icon top right → "New repository".
  2. Name the repo Use ask-kris as the repository name. Set it to Public. Do NOT initialise with a README — we're pushing our own files.
  3. Click "Create repository" GitHub shows you the empty repo page. Leave it open — you'll need the repo URL in a moment.

2 Upload the files to GitHub

The easiest method — no command line needed:

  1. On your empty repo page, click "uploading an existing file" It's a link in the middle of the page — "…or push an existing repository from the command line". Look for the "uploading an existing file" link just above that.
  2. Drag and drop the entire ask-kris folder contents Open the unzipped ask-kris-deploy folder on your computer. Select ALL files and folders inside it (index.html, README.md, netlify.toml, chatbot/, qotd/) and drag them into the GitHub upload area.
  3. Commit the files Scroll down, leave the commit message as "Initial commit", click Commit changes. GitHub uploads and commits everything.
  4. Verify the structure You should see chatbot/ and qotd/ as folders in your repo, plus index.html, netlify.toml, and README.md at the root.
⚠️
GitHub's drag-and-drop uploader handles subfolders correctly as long as you drag the folder itself (not just its contents). If chatbot/ doesn't appear as a folder, try dragging the chatbot folder directly onto the upload area.

3 Connect Netlify to GitHub

  1. Log in to app.netlify.com Go to your Netlify dashboard.
  2. Click "Add new site" → "Import an existing project" Then click GitHub as the Git provider.
  3. Authorise Netlify to access GitHub if prompted Grant access to your repositories.
  4. Search for and select your "ask-kris" repo It should appear in the list. Click it.
  5. Configure build settings Leave everything as defaults — no build command, publish directory is / (root). Netlify detects the netlify.toml automatically. Click Deploy site.
  6. Wait ~30 seconds for the first deploy Netlify shows a green "Published" badge when done. You'll get a URL like random-name-123.netlify.app.

4 Rename your Netlify site

  1. Go to Site configuration → General → Site details Click Change site name.
  2. Enter a memorable name Try ask-kris or kris-haskins-tools. This gives you a clean URL like ask-kris.netlify.app.

5 Set the Apps Script URL

Now that the site is live, update both HTML files with your real API URL and push to GitHub — Netlify redeploys automatically.

  1. Open chatbot/index.html on your computer Find this line near the top of the script section and replace the placeholder:
const API_URL = 'https://script.google.com/macros/s/YOUR_ID/exec';
  1. Do the same in qotd/index.html Same line, same replacement.
  2. Push the updated files to GitHub Go to your GitHub repo → click chatbot/index.html → click the pencil ✏️ edit icon → paste the updated content → commit. Repeat for qotd/index.html.
  3. Netlify auto-redeploys in ~30 seconds Every push to main triggers a new deploy automatically. No manual steps needed.

6 Verify everything works

  1. Visit your live landing page https://your-site.netlify.app/ — you should see the navy landing page with links to both tools.
  2. Test the chatbot /chatbot/ — submit the lead gate, ask "How does Section 8 work?". You should get a real AI response with a product card.
  3. Test the QOTD page /qotd/ — the demo question should load, answer options should be clickable, and the reveal should work.
  4. Check the Leads sheet After submitting the chatbot lead gate, a new row should appear in the Google Sheets Leads tab.
✅
You're live! Share your-site.netlify.app/chatbot/ as the Ask Kris link and your-site.netlify.app/qotd/ as the QOTD link. Update the QR code flyer with the live QOTD URL.

→ Making future updates

Any time you need to update the chatbot or QOTD page — just edit the file on GitHub and commit. Netlify picks up the change and redeploys in under a minute. No FTP, no manual uploads.

💡
Once the CMS (Step 5) is built, Kris will never need to touch these files directly — all content changes go through the CMS which writes to Google Sheets, and the live pages pull from there automatically.
